## Overview

`/commit` generates commits in the Agent007 pipe-delimited format. It surveys changes, stages the right files, writes the commit message, and verifies the result.

```
/commit
```

No arguments needed ‚Äî `/commit` reads the current git state and guides you through the process.

---

## Commit Format

```
Tipo|IdTarea|YYYYMMDD|Descripci√≥n breve
```

| Component | Description |
|-----------|-------------|
| `Tipo` | Nature of the change (see types below) |
| `IdTarea` | Ticket / task board ID (omit if none ‚Äî leave empty between pipes) |
| `YYYYMMDD` | Commit date ‚Äî runs `date +%Y%m%d` automatically |
| `Descripci√≥n` | Action in English, imperative mood, max 60 characters |

### Valid Types

| Type | When to use |
|------|-------------|
| `feat` | New functionality |
| `fix` | Bug correction |
| `refactor` | Code restructuring / improvement |
| `review` | Code review adjustments |
| `test` | Tests implementation or adjustment |
| `docs` | Documentation changes |
| `chore` | Maintenance or minor tasks |

## Process

### Step 1 ‚Äî Survey changes

```bash
git status              # Untracked files
git diff --staged       # What's staged
git diff                # Unstaged changes
git log --oneline -5    # Recent commits (match style)
date +%Y%m%d            # Get today's date
```

### Step 2 ‚Äî Stage relevant files

Prefers specific files over `git add -A` to avoid accidentally including secrets or large binaries.

Never stages: `.env`, `secrets/`, credential files, large binaries.

### Step 3 ‚Äî Write commit message

```bash
git commit -m "$(cat <<'EOF'
feat|LIQUI|20260329|Implement complete CRUD for cargos module
EOF
)"
```

### Step 4 ‚Äî Verify

```bash
git log --oneline -1    # Confirm commit exists
git show --stat HEAD    # Confirm correct files included
```

## Pre-Commit Guard

The `pre-commit-guard.py` hook (registered in `settings.json`) intercepts every `git commit` and blocks:

- Staged `.env` files
- Hardcoded secrets (`API_KEY=`, `SECRET=`, `PASSWORD=`, `TOKEN=` with literal values)
- JS `debug` statements (`debugger` keyword)
- Commits that don't follow the pipe-delimited format

The `block-no-verify.py` hook blocks all `git commit --no-verify` calls ‚Äî bypassing the guard is not allowed.

## Pre-Commit Checklist

- Format: `Tipo|IdTarea|YYYYMMDD|Descripci√≥n`
- Description is clear and concise (max 60 characters)
- Code is tested
- No linting errors
- No secrets or credentials in diff

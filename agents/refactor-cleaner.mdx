## Risk Classification

| Classification | Examples | Removal approach |
|---|---|---|
| **SAFE** | Unused imports, unused local variables | Auto-remove after approval |
| **CAREFUL** | Unused exports (may be used externally or via dynamic import) | Remove with review |
| **RISKY** | Unused files/modules (could be entry points, lazy-loaded) | Only with explicit approval |

---

## Hard Constraints

- **Risk classification before ANY removal** â€” never remove without classifying first
- **Batch order**: SAFE â†’ verify tests â†’ CAREFUL â†’ verify tests â†’ RISKY (explicit approval only)
- **Commit after each batch**: `REFACTOR|CLEANUP|YYYYMMDD|Remove [batch] dead code`
- **NEVER remove**:
  - Any `*.test.*`, `*.spec.*`, `tests/`, `test/` files
  - Config files: `*.config.ts`, `tsconfig*.json`, `.eslintrc*`, `.prettierrc*`
  - Documentation files: `*.md`, `*.mdx`, `docs/**`
  - Migration files: `**/migrations/**`
  - Seed or fixture files
- **ALWAYS verify tests pass** after each batch before proceeding
- **PRESENT findings** to user before acting â€” never auto-remove CAREFUL or RISKY items

## Detection Phase

```bash
# TypeScript unused locals (fast, no tooling needed)
npx tsc --noUnusedLocals --noUnusedParameters --noEmit

# knip (best for unused exports + files)
npx knip --reporter json

# depcheck (unused packages)
npx depcheck --json

# Manual grep fallback
grep -r "export function\|export const\|export class" src/ --include="*.ts" -l
```

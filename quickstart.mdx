## Prerequisites

- [Claude Code](https://claude.ai/code) installed (CLI, desktop app, or IDE extension)
- A project directory (any codebase — language-agnostic)
- Python 3 available on PATH (all hooks are Python; no bash deps)

---

## Installation

Agent007 ships through the official Claude Code marketplace.

<Steps>
  <Step title="Add the marketplace">
    ```bash
    /plugin marketplace add SebastianDevps/agent007-marketplace
    ```
  </Step>

  <Step title="Install Agent007">
    ```bash
    /plugin install agent007@agent007-marketplace
    ```

    This drops a `.claude/` directory into your project root:

    ```
    your-project/
    ├── .claude/
    │   ├── CLAUDE.md          ← orchestrator identity (auto-loaded)
    │   ├── CONTEXT.md         ← project navigation map (auto-loaded)
    │   ├── settings.json      ← hook registration + permissions
    │   ├── agents/            ← 8 expert agents
    │   ├── commands/          ← 8 entry commands
    │   ├── skills/            ← 44 skills (core, pipeline, domain, etc.)
    │   ├── hooks/             ← 26 hooks (Python 3)
    │   ├── rules/             ← 5 coding rule files
    │   └── scripts/
    │       ├── lifecycle/     ← verify, install, uninstall, sync, test, waste
    │       ├── instinct-engine.js
    │       ├── security-scan.js
    │       └── wave-scheduler.js
    └── your-project-files...
    ```
  </Step>

  <Step title="Open Claude Code">
    Start a session in your project. Claude Code automatically loads `CLAUDE.md` + `CONTEXT.md`.

    On your first message you'll see the routing classification:
    ```
    🎯 feature | Risk: low
    ```
  </Step>

  <Step title="Run your first command">
    ```
    /dev "Add a /health endpoint returning service uptime"
    ```

    Agent007 classifies the task, routes it (trivial → inline, substantial → SDD), dispatches the right expert, and verifies output before claiming done.
  </Step>
</Steps>

### Updates

```bash
/plugin update agent007
```

### Verifying the install

From the project root, run the lifecycle verification script:

```bash
.claude/scripts/lifecycle/verify.sh
```

5 checks (+ optional 6th: hook regression tests). Mirror of the CI gate that runs on every plugin PR.

## Your first 3 commands

### 1. Ask an expert

```
/consult "Should I use cursor-based or offset pagination for 10M records?"
```

Auto-routes to `backend-db-expert` (opus model). Returns a structured recommendation with trade-offs and an implementation plan. Optional `--quick` (≤25 lines) or `--deep` (≤120 lines, full skills).

### 2. Run a task autonomously

```
/dev "Add email verification to the registration flow" --ralph --verify "npm test"
```

Full pipeline: classify → SDD if substantial → subagent execution → loop until `npm test` passes or `loop-operator` escalates.

### 3. Orchestrate a multi-agent workflow

```
/orchestrate feature "Build the notifications module"
```

Runs the chain: `brainstorm → plan → SDD → code-reviewer → security-expert → branch-finish`. Each agent emits a structured `HANDOFF.md` to the next. Final report ships SHIP / NEEDS WORK / BLOCKED with a verdict.

## Key concepts

<AccordionGroup>
  <Accordion title="Trivial vs Substantial pipeline">
    The orchestrator decides once per request:

    - **Trivial** (single file, no new behavior, no public surface change) → `Skill('generate')` → `Skill('verify')`
    - **Substantial** (new behavior, multi-file, public surface, refactor, OR any high/critical risk) → `/sdd-new <change>` (proposal → spec → design → tasks → apply → verify → archive)

    When in doubt, choose SDD. Over-planning a small change costs less than under-planning a substantial one.
  </Accordion>

  <Accordion title="Evidence-gated verification">
    Agent007 never claims "done" without running the verification command and showing the actual output:

    ```
    Verified: npm test → 47 passed, 0 failed (2.3s)
    ```

    Phrases like "should work", "probably", "I assume" are blocked by `core/banned-phrases` and replaced with evidence. See [skills/core/banned-phrases.md] for the exact list.
  </Accordion>

  <Accordion title="Frontend anti-convergence gate">
    The `frontend-discovery-gate` hook (PreToolUse/Edit|Write) blocks any write to `.tsx/.jsx/.css/.html/.svelte/.vue/.astro` unless `.claude/state/discovery-output.json` exists and is less than 30 minutes old.

    To unlock, run `Skill('discovery-before-code')` first: referent fetch → 1 of 11 styles → states table → tokens. The gate is in the tool layer — the LLM cannot route around it.
  </Accordion>

  <Accordion title="Ralph loop">
    Autonomous iteration engine. Runs until the `<promise>COMPLETE</promise>` signal is detected.

    ```
    /ralph-loop "Fix all TypeScript errors" --max-iterations 10 --verify "npx tsc --noEmit"
    ```

    The `loop-operator` agent watches for stalls (same error × 3), cost drift (> 2× baseline), and context budget (> 80%). It escalates with A/B/C recovery options instead of looping silently.
  </Accordion>

  <Accordion title="Tool policy per agent (least privilege)">
    Each agent only sees the tools it needs:

    | Profile | Agents | Tools |
    |---|---|---|
    | `minimal` | security-expert, product-expert, code-reviewer (+ Bash read-only) | Read, Glob, Grep |
    | `coding` | backend-db-expert, frontend-ux-expert, platform-expert, refactor-cleaner | + Write, Edit, Bash, Agent (frontend also: WebFetch, WebSearch) |
    | `full` | loop-operator | everything |

    The `tool-policy-guard` hook (PreToolUse/Write\|Edit) enforces this at runtime — a misrouted agent is blocked before it can write.
  </Accordion>
</AccordionGroup>

## Common workflows

### Bug fix (direct)
```
/dev "Fix null pointer in UserService.findById()" --simple
```

### New feature (medium)
```
/dev "Add cursor-based pagination to /users endpoint"
```

### Complex feature (full pipeline + ralph)
```
/dev "Implement JWT auth with refresh token rotation" --full --ralph --max-iterations 40 --verify "npm run test:e2e"
```

### Visual work (anti-convergence enforced)
```
/dev "Build a pricing page that looks like linear.app/pricing"
```
The discovery skill fires first. Without a recent discovery output, the hook blocks every `.tsx`/`.css` write.

### Security audit
```
/consult "Review the auth module at src/auth/" --experts security
```

### Dead code cleanup
```
/orchestrate cleanup "Remove all unused exports and dependencies"
```

### Vague intent → executable prompt
```
/consult "How should I structure bounded contexts for payments?"
/prompt-gen "design payments module with bounded contexts" --target dev --save
```

`/prompt-gen` produces a canonical Anthropic-conventions XML spec persisted at `.sdlc/state/active-prompt.json`, then `subagent-context.py` injects it into every delegated subagent.

## Token budget tips

| Mode | Approx tokens | When |
|---|---|---|
| `/consult --quick` | ~2K | Direct answer, no trade-offs |
| `/consult` | ~5K | Analysis with trade-offs |
| `/consult --deep` | ~10K | Architecture decisions |
| `/dev --simple` | ~3–8K | Bug, single-file change |
| `/dev` medium | ~15–40K | Feature, 3–6 tasks |
| `/dev --full` | ~40–80K | Complex feature with brainstorm |

Use `/compact` between phases (after plan, before implementation). **Never** in the middle of a `subagent-driven-development` run — it loses task state.

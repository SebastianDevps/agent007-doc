## What is Agent007?

Agent007 is a Claude Code plugin that transforms how you develop software. It provides an **intelligent orchestration layer** on top of Claude — a team of 8 expert agents, 25 active skills, and 12 commands that work together to plan, implement, review, and ship production-quality code autonomously.

Every message is **automatically classified and routed** to the right expert. Every claim is **backed by evidence**. Every task follows the **SDD Iron Law**: no production code without a defined observable scenario first.

## Routing Intelligence

Every message is classified before any action is taken:

```
🎯 [TYPE] → [ROUTE] | Risk: [low|medium|high|critical]
```

| Message type | Signal words | Route |
|---|---|---|
| feature | add, implement, create, build | `/dev` |
| bug | fix, error, broken, not working | systematic-debugging → SDD |
| consult | should I, recommend, best practice | `/consult` |
| refactor | refactor, clean up, optimize | `sop-reverse` → plan |
| review | code review, audit, check quality | `code-reviewer` |
| research | investigate, explore, compare | `/deep-research` |
| ralph | loop until, autonomous, until tests pass | `loop-operator` → `/ralph-loop` |
| cleanup | dead code, unused, depcheck | `refactor-cleaner` |

## Immutable Rules

These rules are always active and cannot be overridden:

- **NEVER** claim "done" without `Skill('verify')` passing with evidence
- **NEVER** write production code before defining an observable scenario (SDD Iron Law)
- **NEVER** modify scenarios to make code pass — scenarios are external holdout sets
- **NEVER** assume user approval — require explicit confirmation
- **ALWAYS** read a file before editing it
- **ALWAYS** show routing decision before any skill invocation

## Pipeline Map

```
simple  → generate → verify → done
          On FAIL: retry with feedback (max 3). On 3× FAIL: escalate.

medium  → plan → [generate → verify] → code-reviewer → branch-finish

complex → brainstorming → git-worktree → plan
          → [generate → verify] + ralph-loop per task
          → human gate → branch-finish

refactor → sop-reverse → plan → medium pipeline
bug      → systematic-debugging → generate → verify
```

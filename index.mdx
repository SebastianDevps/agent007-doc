## What is Agent007?

Agent007 is a Claude Code plugin that turns the base CLI into an opinionated development orchestrator. It classifies every request, routes it to the right expert, runs deterministic safety hooks on every tool call, and never claims "done" without showing real evidence.

The plugin is designed for one thing: **shipping production code without watching the agent every second.** Routing happens at the tool level, not in a checklist the model might forget. Every visual file write is gated by an anti-convergence hook. Every commit goes through a pre-commit guard. Every long loop is bounded by a stall detector.

## Frontend builder gate (the v6 anti-convergence story)

```
.claude/state/discovery-output.json
 ↑ written by Skill('discovery-before-code')
 ↓ read by frontend-discovery-gate hook (PreToolUse/Edit|Write)

If output is missing or older than 30 minutes → write to .tsx/.css is BLOCKED.
```

The skill enforces 4 steps before any visual code:

<Steps>
  <Step title="Referent fetch">
    A real URL (not "modern dashboard"). Documented with `WebFetch` and recorded in the discovery output.
  </Step>
  <Step title="Pick 1 of 11 extreme styles">
    brutalist · cyberpunk · swiss · memphis · neumorphism · glassmorphism · skeuomorphic · y2k · editorial · monospace-terminal · risograph. Anti-convergence by construction.
  </Step>
  <Step title="States table">
    empty · loading · error · success · edge — designed before any component is written.
  </Step>
  <Step title="Tokens declared">
    ≤5 colors, ≤2 fonts. Then executable skills take over: `shadcn-component-install`, `design-tokens-extract`, `a11y-contrast-check`, `page-transitions-barba`, `ios-hig-mobile`, `spline-3d-embed`, `design-system-doc`.
  </Step>
</Steps>

## Routing intelligence

Every message is classified before any action. The orchestrator announces:

```
🎯 [target] | Risk: [low|medium|high|critical]
```

| Intent | Signal words | Route |
|---|---|---|
| feature | add, implement, create, build | `/dev` |
| bug | fix, error, broken, not working | `systematic-debugging` skill |
| consult | should I, recommend, best practice | `/consult` |
| refactor | refactor, clean up, optimize | `sop-reverse` skill |
| review | code review, audit, check quality | `code-reviewer` agent |
| research | investigate, explore, compare | `/deep-research` skill |
| ralph | loop until, autonomous, until tests pass | `loop-operator` → `/ralph-loop` |
| cleanup | dead code, unused, depcheck | `refactor-cleaner` agent |

High/critical risk (auth, payments, encryption, migrations, breaking changes) requires explicit "yes" before acting.

## Hard rules

These are enforced by hooks, not by prompt instructions:

- **NEVER** claim "done" without `Skill('verify')` showing actual command output (`[cmd] → [output]`)
- **NEVER** write production code before defining an observable scenario (SDD Iron Law)
- **NEVER** modify test scenarios to make code pass — scenarios are external holdout sets
- **NEVER** commit `.env` files, hardcoded secrets, or use `git --no-verify`
- **NEVER** assume user approval for high-risk work — explicit "yes" required
- **ALWAYS** read a file before editing it
- **ALWAYS** show routing decision before any agent or skill invocation

## Pipeline map

```
Trivial      → Skill('generate') → Skill('verify') → done
Substantial  → /sdd-new → proposal → spec → design → tasks → apply → verify → archive
Bug          → systematic-debugging → reproduce → fix → verify
Refactor     → sop-reverse → plan → SDD pipeline
Visual code  → discovery-before-code (BLOCKING gate) → tokens → component
```

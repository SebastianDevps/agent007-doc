## [6.0.8] — 2026-05-07 — "Frontend Builder + CI gate"

> Major refactor. Plugin now self-validates via a 6-job GitHub Actions workflow. Eager-loaded line count reduced 84%.

### Added — frontend execution

- **`frontend-ux-expert` is now a BUILDER** — was previously read-only (Read/Grep/Glob). Now ships code by default with `[Read, Write, Edit, Glob, Grep, Bash, WebFetch, WebSearch]`. Four explicit modes:
  - `BUILDER` (default) — writes code, not opinions
  - `PLANNER` — wireframes + ordered tasks, no implementation
  - `CONSULTANT` — recommendation + component API sketch, no implementation
  - `REVIEWER` — only when user explicitly asks; never the default
- **`frontend-discovery-gate.py`** (PreToolUse/Edit|Write) — blocks writes to `.tsx/.jsx/.css/.html/.svelte/.vue/.astro` if `.claude/state/discovery-output.json` is missing or older than 30 minutes. Anti-convergence enforced at the tool layer, not by prompt.
- **8 new actionable frontend skills** — all gated by `discovery-before-code`:
  - `discovery-before-code` — referent + 1 of 11 styles + states + tokens (BLOCKING)
  - `shadcn-component-install` — info → search → docs → diff → install (mandatory dry-run)
  - `a11y-contrast-check` — Node.js zero-deps WCAG checker, exits 1 if <4.5:1
  - `design-tokens-extract` — WebFetch referent → write `tokens.css` / `tokens.json` (W3C format)
  - `design-system-doc` — canonical 9-section DESIGN.md
  - `page-transitions-barba` — barba.js + GSAP for Vite/Next Pages/Astro/plain (forbidden in Next App Router)
  - `ios-hig-mobile` — mobile-first components following Apple HIG
  - `spline-3d-embed` — Spline 3D scenes lazy-loaded, perf-gated, mobile-aware

### Added — defensive guards

- **`path-existence-guard.py`** (PreToolUse/Edit|Write|Read) — blocks operations on hallucinated paths
- **`tool-allowlist-guard.py`** (PreToolUse/Bash) — skill-level bash whitelist (shadcn-style pattern)
- **`context-tick.py`** (SessionStart + PostToolUse + Stop) — persists telemetry to `.sdlc/state/context-budget.jsonl`
- **`session-recover.py`** (SessionStart) — emits previous-session preamble if last activity < 4h ago

### Added — lifecycle and CI

- **6 lifecycle scripts** in `.claude/scripts/lifecycle/`:
  - `verify.sh` — 5 checks (+ optional 6th: hook regression tests)
  - `install.sh /path/to/proj` — deploy to a project
  - `uninstall.sh /path/to/proj` — remove (preserves state)
  - `sync-to-public.sh` — sync to marketplace
  - `test-hooks.py` — 13 fixture-based regression tests
  - `waste-report.py` — telemetry audit: top files, hit rate, p95 tokens
- **CI gate** at `.github/workflows/plugin-validate.yml` — same 5 checks on every PR
- **`.line-cap-exemptions`** debt register — currently empty (zero eager-loaded files > 200 lines)

### Changed

- **Eager-loaded total reduced 84%** (~7300 → 1195 lines). Auto-inject overhead reduced 81% (1239 → 237 lines per session).
- **`CLAUDE.md` rewritten** as identity + core rules + routing only. Project navigation moved to `CONTEXT.md` (lazy-loaded).
- Skill registry now uses frontmatter contracts: `allowed-tools`, `canonical-sources`, `references`, `when.keywords`.
- Memory protocol uses 3-layer disclosure (Engram): `mem_search` → `mem_timeline` → `mem_get_observation`. ~10× token savings vs single-fetch.
- `topic_key` convention enforced (`<domain>/<area>/<sub>`). Temporal validity via `superseded_by:{old_id}`.

### Removed

- `architect` agent — duplicated `architecture-patterns` skill, no handoff routes. Replacement: `Skill('architecture-patterns')`.
- `performance-optimizer` agent — extracted into `quality-gates/performance-profiling/` skill (3 layer-specific protocols).

## [5.0.0] — 2026-04-02 — "OpenClaw Integration"

### Added

**10 OpenClaw primitives** — orchestration and safety hooks:

| Primitive | Hook event | Behavior |
|---|---|---|
| `tool-loop-detection` | PostToolUse/all | SHA-256 fingerprint window (30 calls). Warning at 10, circuit breaker at 30. |
| `context-engine` | PreToolUse/Agent + Stop | Token budget estimation. Writes `context-budget.json`. Blocks at 80%. |
| `mutation-guard` | PreToolUse/Write\|Edit\|Bash | Content-hash dedup. Skips exact duplicate writes silently. |
| `memory-decay` | SessionStart | Exponential decay (λ=0.023). Stale at 30 days, archived at 60. |
| `tool-policy-guard` | PreToolUse/Write\|Edit | Enforces `tool_profile: minimal/coding/full`. Fail-open if no profile. |
| `provider-rotation` | PreToolUse/Agent | Cooldown 5/15/60 min. Failover: opus → sonnet → haiku. |
| `transcript-policy` | SubagentStart | Model-tier directive: haiku → concise, opus → deep-analysis. |
| **Yield Pattern** | SDD skill v3.0 | Parallel wave execution via multiple `Agent()` calls per response. |
| **Steer Pattern** | loop-operator | Drift guidance via `SendMessage` before kill+restart. Max 2 attempts. |
| **TaskFlow** | session.json | Optimistic locking via `revision`. Lifecycle: idle→execute→wait→resume→finish. |

**Wave Scheduler** — `scripts/wave-scheduler.js`: topological sort (Kahn's algorithm) for task dependency graphs. Outputs parallel execution waves from `tasks.md`.

**Tool policy profiles** added to all 8 agents (minimal / coding / full).

### Changed

- `loop-operator.md` adds Steer Pattern to escalation matrix. Drift → STEER before KILL+RESTART.
- Hook count: 17 → **28** (7 OpenClaw + provider-rotation + transcript-policy registered).

### Removed

- `hooks/ralph-check.js` → replaced by `tool-loop-detection.py`

## [5.1.1] — 2026-03-29 — "ECC Integration (P0 + P1)"

### Added

**Pre-commit quality hook** — `hooks/pre-commit-guard.py` (PreToolUse/Bash) intercepts `git commit`:
- Blocks staged `.env` files and hardcoded secrets (`API_KEY=`, `SECRET=`, `PASSWORD=`, `TOKEN=`)
- Blocks `debugger` statements; warns on `console.log`
- Validates Agent007 pipe-delimited commit message format

Paired with `block-no-verify.py` blocking all `git commit --no-verify` calls.

**91 new integration tests** across 5 suites covering pre-commit guard, hook-chain, state-sync, context-budget, instinct-engine, and wave-scheduler.

**`code-reviewer` agent** — sonnet-model, read-only:
- CRITICAL/HIGH/MEDIUM/LOW severity taxonomy
- 80% confidence filter (issues below threshold dropped)
- Consolidation of similar findings
- AI cost-awareness check (`SKIP_REVIEW` for trivial changesets)

**Tool restrictions per agent** (principle of least privilege) — `tools:` YAML frontmatter on all 8 agents.

**`loop-operator` agent** — autonomous loop controller:
- Pre-start checklist; checkpoint tracking per iteration
- Stall detection (same error × 3); cost drift monitoring (2× baseline)
- Structured escalation with A/B/C recovery options

**`/orchestrate` command** — 5 named workflow types with `HANDOFF.md` protocol:
- `feature` — brainstorm → plan → SDD → code-reviewer → security → branch-finish
- `bugfix` — systematic-debugging → plan → generate → code-reviewer → verify
- `refactor` — sop-reverse → plan → SDD → code-reviewer → verify
- `security` — security-expert → code-reviewer → fix → re-verify
- `cleanup` — refactor-cleaner → verify

**3 new rule files** — `git-workflow.md`, `patterns.md`, `coding-style.md`.

**`refactor-cleaner` agent** — runs `knip`, `depcheck`, `ts-prune`. Three-tier risk: SAFE / CAREFUL / RISKY. Batch removal with test verification + atomic commit per batch.
